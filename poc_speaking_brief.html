디지털마케팅팀 커머스파트 · 이채영 · 2026.07.08 기준

1왜 이 프로젝트를 시작했는가

시장·정책·내부 데이터, 세 방향에서 같은 신호가 확인됐어요.

시장 니즈 — 학부모들은 "영어를 잘하게" 보다 "다음 학령 단계에서 무너지지 않는 것"을 원해요. 유치·초등 저학년까지는 "흥미 형성"이 최우선이지만, 초등 고학년으로 갈수록 "원서 읽기만으로 중등 내신까지 이어질 수 있을지" 걱정하는 목소리로 바뀌고, 중등에서는 "초등 때 영어를 오래 했는데도 내신에서 흔들릴 수 있다"는 실제 불안으로 이어져요.
교육정책 흐름 — 2026 교육감 공약 공통 키워드는 기초학력·맞춤형학습·AI디지털·서술형·수행평가. 2028 대입 개편과 맞물려 "문해력+서술형+수행평가"가 영어교육의 핵심으로 압축되는 흐름이라, 수능 자체보다 학교 내신·세특·수행평가·발표·쓰기 역량의 중요성이 더 커질 걸로 보여요.
내부 데이터 (권용훈 팀장, Times 사업) — 신문:화상영어 매출 비율은 대략 4:6이고, 화상영어 이용자의 70% 이상이 초등학생이고 그중 초2–4학년이 최다. 상품 이용 순위는 키즈 > 주니어 > 킨더 > 타임즈 순. 실제 녹화 수업 영상을 리뷰해보니 초등 고학년 중 이미 free speech가 가능한 학생군이 상당수 확인됐어요.
⚠️ 초5–6을 1차 타겟으로 잡은 이유 — 초2–4에서 화상영어를 시작한 아이가 초5–6이 되며 "다음 콘텐츠가 없어 이탈"하는 구간이에요. 이 서비스가 이탈 방어 + 상향 이용 역할을 할 수 있다는 가설이고, 실제 발화력을 갖춘 학생군에게 구조화된 수행평가 훈련을 제공할 수 있는 시점이기도 해요.
종합 시사점 — 학부모는 지금의 학습이 다음 학년·다음 평가에서 어떤 성과로 이어지는지 눈으로 확인하고 싶어해요. NE능률 입장에서도 읽기·듣기·말하기·쓰기를 따로따로 보여주기보다, 초등부터 중등 내신·고등 수능까지 이어지는 하나의 성장 경로로 제시하는 게 지금 시장·정책 흐름과 가장 잘 맞아요.

2무엇을 만들었나 (목적)

기존 NE Times 화상영어 자산에, 학교 수행평가 대비 스피킹 트레이닝을 얹는 컨셉 — "NE Times 수행평가랩".

  • 실제 NE Times 기사 4건 × 실제 수행평가 4개 유형을 1:1 매칭한 인터랙티브 POC 제작
  • 기사 요약 말하기 / 1분 스피치 / PPT 발표 / 인터뷰 대본 말하기 — 총 4개 트랙
  • 시스템이 단계별로 이끄는 task형 학습 구조 (문장별 3회 듣기·3회 말하기 반복)
  • 원어민 개입은 마지막 5분 코칭으로 최소화, 발음 매칭 엔진(참고용) 및 학습 포트폴리오까지 구현

현재 단계의 목적은 완성도 있는 서비스 출시가 아니라, "이 방향이 추가 개발할 가치가 있는가"를 검증하는 것이에요.

5POC 설계 전제조건

지금까지의 대화에서 확정된, 앞으로도 지켜야 할 설계 원칙들이에요.

전제 1

목표 요약문은 압축버전으로 확정 — 초5–6 수행평가 분량·난이도 기준

전제 2

현장 최종결과물 best sample 정의는 향후 계획으로 보류

전제 3

원어민 개입 최소화 — 기존 화상영어 세션 마지막 5분만 활용

전제 4

창의성은 평가 항목 아님 — 자유 표현은 활동의 일부일 뿐

전제 5

원어민 노출은 항상 1회 고정, 자율연습 횟수만 늘림

전제 6

회차 1–3, 문장별 3회 듣기 + 3회 말하기 기본값

전제 7

기사 1건 = 수행평가 유형 1개 — 중복 배정 없음

전제 8

발음 매칭 점수는 참고용 — 음성인식 기반 단어 일치도

6후속 업무

  1. 수행평가 유형분석 지속 — 시도교육청별 실제 채점기준 확보
  2. 쓰기(Writing) 영역으로 확장NEW — 4트랙 프레임워크를 서술형·에세이 수행평가로 확장
  3. 자사 ELT 교재 시리즈 연계NEW — Speaking Stage, Write Right 시리즈까지 동일 구조 적용
  4. 학습 포트폴리오 고도화 — 실제 데이터 연동 및 학부모 열람 기능 확장
  5. B2C → B2B 확장 로드맵 실행 — 홈스쿨링 파일럿(8–12주) 후 클래스 관리 기능으로 확장

7주요 개발 이력 (IT 개발자 인수인계용)

POC가 지금 형태에 이르기까지 거친 주요 결정과 수정 포인트예요.

① 핵심 데이터 구조

ARTICLES 배열의 각 기사는 genre·line·type 3필드로 정확히 1개 조합만 매칭 (1기사=1유형 원칙). 각 세션은 이전 세션 내용을 포함하는 누적 배열이며, 문장마다 organizer 필드로 그래픽 오거나이저 색상이 결정돼요.

② 학습 진행 단위 재설계

세션 전체 3회 반복에서 → 문장 단위 각각 듣기 3회·말하기 3회를 채우는 itemProgress 배열 구조로 전면 교체. "지난 시간 복습" / "오늘 배울 문장"을 그룹블록으로 명확히 분리했어요.

③ 발견 및 수정된 버그 2건

(a) "학습 시작" 버튼 미반응 — state.screen을 practice로 전환하는 코드 누락. 화면 분기 로직 수정 시 상태값 갱신 여부 필수 확인.
(b) 자유입력 값이 회차 이동 시 삭제 — articleId+문장idx 키로 별도 저장소(freeAnswers)로 분리 해결.

④ TTS 안정화

speechSynthesis.cancel() 직후 speak() 호출 시 소리 미출력 레이스컨디션 → 120ms 지연 삽입. en-US 보이스 직접 선택, onend 미발동 대비 타임아웃 안전장치, 화면 전환 시 재생 큐 취소 처리.

⑤ 발음 매칭 엔진 — "참고용" 라벨 유지 필수

webkitSpeechRecognition으로 실시간 전사 후 단어 단위 일치율(%)만 계산. 실제 음소 단위 발음 평가가 아니에요 — UI 문구에서 이 구분을 절대 제거하지 마세요.

⑥ UI 개편 이력

"장르·유형별 개별 카드+드롭다운" → "3개 필터 동적 결과" → "책 표지 스타일 이미지 그리드(3:4 비율)" 순으로 발전. 실제 이미지 확보 기사만 <img> 렌더링, 실패 시 아이콘으로 자동 폴백(onerror 처리).

⑦ 콘텐츠 규칙: 인터뷰형(kind:'qa') 자유입력 지양

Malala 인터뷰 마지막 질문이 원래 자유 서술형이었으나, "Thank you.../My pleasure." 고정 교환으로 변경 확정. 인터뷰형 트랙은 원칙적으로 완전 자유 응답을 넣지 않음.

⑧ 알려진 리소스 제약

이미지: 로그인 필요 페이지는 이미지 URL 미확보 → 정식 개발 시 NE Times 이미지 CDN 인증 접근 경로 필요.
폰트: DengXian은 마이크로소프트 전용 라이선스 → Noto Sans KR + Inter + Nunito 조합으로 대체.
음성인식: webkitSpeechRecognition은 Chrome 계열에서만 동작.

⑨ 학습 포트폴리오

코칭 rubric 4개 항목이 모두 체크되는 시점에 기록 1건이 자동 로깅돼요. 데모용 샘플 기록 2건이 기본 포함됩니다 — 실서비스 연동 시 이 시드 데이터는 반드시 제거해야 해요.
